## 1. プロジェクト概要

高校理系・大学受験向けの物理カード学習アプリを作る。

目的は、単なる公式暗記アプリではなく、以下を満たすこと。

- 公式を覚える
- 公式の意味を理解する
- 導出・条件・使いどころも確認できる
- 「この問題はどの法則で入るか」という典型判断を鍛える
- 間違えたカード・苦手カードが再出題されやすい
- 高校生の女子でも見やすく、かわいいUIにする
- スマホで使いやすい
- ホーム画面に追加してアプリ風に使える
- GitHub Pagesで公開する
- 重くしすぎず、後から単元を追加できる

## 7. 学習設計

カードは以下の型を重視する。

- 公式
- 条件
- 意味
- 保存則
- 単位
- 典型判断

物理では、公式を知っているだけでは解けない。
ただし公式を知らないと入口にも立てない。

特に重要なのは以下。

- この式はいつ使えるのか
- 何を保存させるのか
- どの物体に着目するのか
- 電場なのか電位なのか
- 回路ならどこで電流保存、どこで電圧保存か
- この問題はどの法則で入るのか

## 9. Google Drive保存方針

旧Driveデータは無理に消さない。

理由:

- `appDataFolder` は通常のDrive画面から見えず、手動削除が難しい
- 進捗JSONは容量が小さい
- 旧データが保険になる

v2では旧版と保存名を分ける。

```text
physics-g-cards-v2-progress.json
```

これにより、

- 旧データは残る
- v2は旧データを読まない
- v2用に新しく保存される

という安全な状態にする。

## 10. PWA / Service Worker 方針

PWA対応は便利だが、開発中はService Workerがキャッシュ事故を起こしやすい。

方針:

- 開発中はService Workerを弱める、または一時的に外す
- 安定後にPWA対応を戻す
- キャッシュ名は更新ごとに必ず変える
- activate時に古いキャッシュを削除する
- ホーム画面アイコンは、古い版を削除してから追加し直す

例:

```js
const CACHE_NAME = "physics-g-cards-v2-1-0";
```

更新時は `v2-1-1`、`v2-1-2` のように変える。

## 13. 現在の最重要未解決課題: 数式表記

子供から「公式が見にくい」と指摘があった。

問題:

- `F = G Mm / r^2` のような斜線分数が見にくい
- 分母分子が斜めに区切られる表記に違和感がある
- 教科書のような縦分数でないと直感的に入りにくい
- 公式集タブでは特に違和感が大きい

方針:

**数式を教科書風表記に直す。**

実装候補:

- KaTeXを使う
- 主要公式はLaTeX文字列で管理する
- 公式集タブでは縦分数・添字・指数を徹底する
- カード本文の主要式も順次KaTeX表示にする

例:

```text
F = G Mm / r^2
```

ではなく、

```latex
F = G \frac{Mm}{r^2}
```

として、縦分数で表示する。

## 14. UI方針

女子高校生にも使いやすい、かわいく柔らかいUI。

方向性:

- 淡いピンク
- 淡い水色
- 丸みのあるカード
- きつすぎない影
- スマホで読みやすい文字サイズ
- 公式は大きく独立表示
- 本文と公式のコントラストを確保
- 背景は学習を邪魔しない程度に薄く

背景:

- ユーザー提供のバイク写真をイラスト風にしたものを使う
- ただし淡く、学習画面の邪魔にならないようにする
- カード本文の背後は白系または半透明パネルにする

アイコン:

- バイクを中心
- 周囲に物理公式
- 小さく表示しても潰れないことを優先
- 公式は飾りとして扱い、読ませすぎない

候補公式:

- `F = ma`
- `v = v₀ + at`
- `E = mc²`
- `V = IR`
- `W = qV`
- `p = mv`
- `E = \frac{1}{2}mv^2`
